Ти senior Python developer. Допоможи доробити MVP-проєкт Telegram-бота для моніторингу квартир на Sreality.cz та Bezrealitky.cz.

Ціль:
- бот має перевіряти нові оголошення квартир;
- параметри: Praha 5, Praha 13, Stodůlky;
- розмір/планування: від 3+kk і більше, також 3+1, 4+kk, 4+1, 5+kk;
- як тільки з’являється нове оголошення — відправляти його в Telegram;
- бот має працювати без власного сервера через GitHub Actions;
- не спамити старими оголошеннями;
- зберігати вже знайдені оголошення в `data/seen.json`.

Вимоги:
1. Python 3.12.
2. Використовуй requests, BeautifulSoup, PyYAML, python-dotenv.
3. Не роби агресивний scraping:
   - нормальний User-Agent;
   - timeout;
   - пауза між запитами;
   - запуск не частіше ніж раз на 10–15 хвилин.
4. Telegram credentials мають братися з env:
   - TELEGRAM_BOT_TOKEN
   - TELEGRAM_CHAT_ID
5. URL пошуку мають бути в `config.yaml`.
6. Якщо сайт змінив HTML — код має не падати повністю, а писати помилку по конкретному source і переходити до наступного.
7. Додай нормальне логування.
8. Покращи парсери:
   - `parse_sreality_html`
   - `parse_bezrealitky_html`
9. Зроби так, щоб фільтр був надійний:
   - враховував title, location, layout, area, url;
   - не відкидав оголошення тільки через те, що локація не виведена в картці, якщо URL джерела вже фільтрує по локації.
10. Перевір, чи GitHub Actions workflow коректно комітить оновлений `data/seen.json`.

Бажаний результат:
- стабільний MVP;
- чистий код;
- README з інструкцією;
- можливість локального запуску;
- можливість запуску через GitHub Actions.
